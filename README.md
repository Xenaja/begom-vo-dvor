# Бегом во двор — лендинг + мини-CRM

Дворовые подвижные игры для детей 4–9 лет: инструктор приходит во двор и проводит
45-минутные активные игры. Проект = маркетинговый лендинг + мини-CRM с расписанием,
онлайн-записью и оплатой.

## Архитектура

```
Браузер ──fetch──► Worker «begom-api» ──► D1 (SQLite «begom»)
  │ index.html          │ /api/*                 таблицы расписания,
  │ (GitHub Pages)      │ материализация,        клиентов, броней, оплат
  └ виджет записи       └ запись/оплата          (см. api/schema.sql)
```

- **Фронт** — статический `index.html` (инлайн CSS/SVG), хостинг GitHub Pages.
- **Бэк** — Cloudflare Worker `begom-api` (`api/`), база Cloudflare **D1**.
- **Крон** — ежедневно (`0 3 * * *`) генерирует занятия из шаблонов расписания на 28 дней вперёд.
- Стек один на все уровни CRM (L1→L3), миграций между уровнями не планируется.
  Бот-кабинет (L3) — на вебхуках TG/VK, без постоянного процесса.

## Структура репозитория

- `index.html` — лендинг + живой виджет записи (расписание → выбор слота → форма). Hero — зелёная «обложка».
- `index-classic.html` — резервная версия лендинга со старым hero (текст слева / дети справа).
- `assets/` — изображения (логотип, иллюстрации детей, дудлы).
- `api/` — бэкенд:
  - `src/index.js` — Worker (роутинг, материализация, расписание, запись).
  - `schema.sql` — схема БД (14 таблиц, сгруппированы по фазам L1/L2/L3).
  - `seed.sql` — реальное расписание июля 2026 (локации, пробные, шаблоны).
  - `wrangler.toml` — конфиг Worker (биндинг D1, крон, переменные).
  - `README.md` — команды разработки и деплоя бэкенда.
- `design_handoff_begom_vo_dvor/` — дизайн-хэндофф (спецификация + референс).
- `visual-elements/` — исходные графические элементы.

## API (кратко)

| Метод | Путь | Назначение |
|---|---|---|
| GET  | `/api/health` | проверка живости |
| GET  | `/api/schedule` | локации + занятия с остатком мест (`free`) |
| POST | `/api/materialize` | сгенерировать занятия из шаблонов (идемпотентно) |
| POST | `/api/book` | запись на занятие |

`POST /api/book` принимает `{session_id, parent_name, phone, child_name, child_age,
consent_pd, consent_offer}`. Оба согласия обязательны. Пробные (цена 0) пишутся сразу
с атомарным резервом места; регулярные (350 ₽) пока возвращают `501` — ждут оплаты Т-Банка.

## Данные

- Деньги — в **копейках** (`INTEGER`), 350 ₽ = `35000`.
- Время — **ISO8601 UTC** (`TEXT`), на фронте показывается в МСК (`Intl`, `Europe/Moscow`).
- Остаток мест **не хранится**, а считается: `capacity − активные брони`
  (бронь `hold` не истёкшая | `paid` | `attended`). Защита от овербукинга — атомарной
  вставкой брони (`INSERT … WHERE count < capacity`).
- «Хребет» — таблица `clients` (телефон-ключ + привязки `tg_id`/`vk_id` + кошелёк + согласия).

## Уровни CRM (дорожная карта)

- **L1** — локации, занятия, вместимость, остаток мест, записи, простая админка.
- **L2** — роль тренера, посещаемость, расчёт ЗП, перенос оплаты (баланс), ручная запись, лиды/сегменты.
- **L3** — авторассылки по сегментам, рефералка, абонементы, мини-кабинет в боте (мои записи/баланс/расписание).

Команды разработки бэкенда — в `api/README.md`.
