# Handoff: Hero-блок «Бегом во двор»

## Overview
Hero-секция для лендинга детской секции подвижных игр «Бегом во двор». Задача блока — представить услугу (подвижные игры на улице рядом с домом), показать цену/формат и вести к целевому действию «Записать ребёнка». В комплекте две версии одной композиции: десктоп (широкая, текст слева / сцена справа) и мобайл (вертикальная, сцена сверху / текст снизу).

## About the Design Files
Файлы в этом бандле — **дизайн-референс, созданный в HTML** (прототип, показывающий задуманный вид и поведение), а не продакшн-код для прямого копирования. `Hero варианты.dc.html` — это «Design Component», который использует рантайм `support.js` (нестандартный шаблонный движок) — **не тащите его в реальный проект**. Задача: **воссоздать эту вёрстку в среде целевого кодовой базы** (React, Vue, Svelte, Next и т.п.) её собственными паттернами и компонентами. Если среды ещё нет — выбрать подходящий фреймворк и реализовать там. Всё, что нужно для реализации, описано ниже; сам HTML нужен лишь как визуальная сверка.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы и композиция. UI следует воссоздать пиксель-в-пиксель средствами кодовой базы. Единственное исключение — фоновые «холмы», круг и волны нарисованы инлайн-SVG; их можно реализовать любым удобным способом (SVG, clip-path, маски).

## Screens / Views

### 1. Hero — Desktop
- **Purpose**: первый экран лендинга; донести оффер и привести к CTA.
- **Layout**: карточка со скруглением 34px, `overflow:hidden`, тень `0 30px 70px rgba(90,110,50,.16)`. Референсный размер контейнера 1180×600px (в проде — адаптивная ширина, соотношение примерно 2:1). Внутри — CSS Grid из 2 колонок `grid-template-columns: .82fr 1.18fr` (левая — текст, правая — сцена).
- **Фон карточки**: `linear-gradient(120deg,#FFFFFF 0%,#FFFFFF 46%,#F3F8E8 100%)`.

**Левая колонка (текст)** — `align-self:center`, паддинги `0 24px 0 64px`:
- **Звёзды** (декор): контейнер 40×40 с большой звездой (40×40, `fill #FCC21B`, обводка `#2B2A26` 1.4px) и маленькой звездой (16×16, тот же стиль), позиционированной абсолютно `top:-8px; right:-40px`, с плавающей анимацией. `margin-bottom:26px`.
- **H1**: «Подвижные игры на улице рядом с домом». Nunito, weight 600, 39px, line-height 1.14, color `#2B2A26`, `text-wrap:balance`, `margin-bottom:16px`.
- **Подзаголовок (p)**: «350 ₽ · 45 минут · 5–6 и 7–9 лет». Nunito, weight 600, 17px, color `#7C7A72`, `margin-bottom:26px`.
- **Плашка (pill)**: «Пробные занятия». inline-flex, gap 10px; фон `#FFF3D6`, бордер `1px solid #F2D58A`, текст `#8A6314` weight 700 15px; паддинг `10px 20px`; radius 999px; слева оранжевая точка 10×10 круглая `#F0A93C`. `margin-bottom:26px`.
- **CTA-кнопка**: «Записать ребёнка». Фон `#7CB342`, текст белый weight 900 20px; паддинг `17px 40px`; radius 999px; тень `0 14px 30px rgba(124,179,66,.4)`.

**Правая колонка (сцена)** — `position:relative; overflow:visible` (важно: `visible`, чтобы фото могло заходить влево на белую область за текстом):
- **Холмы** (фон, z-index 0): два наложенных SVG-пути мягких волн. Верхний слой `#EAF3D6`, нижний `#DCEBBF`.
- **Фото детей + надпись** (z-index 2): `assets/kids-logo.png`. Позиция: `left:40%; bottom:0; transform:translateX(-50%); width:118%`. Смещено левее центра намеренно, чтобы девочка справа полностью помещалась, а мальчик заходил на белую зону слева.
- **Две звезды** (декор, z-index 3): одна 22×22 `top:40px; right:64px`, вторая 16×16 сверху-справа; стиль как у левых звёзд, с анимацией покачивания.
- Белый круг за надписью **вшит в сам PNG** — отдельного элемента нет.

### 2. Hero — Mobile (≈390px)
- **Purpose**: то же, вертикальная раскладка.
- **Layout**: карточка ширина 390px, radius 34px, `overflow:hidden`, фон `#FFFFFF`, высота по контенту (`flex column`), тень как у десктопа.

**Сцена (сверху)** — `position:relative; height:372px; overflow:hidden; background:#EAF3D6`:
- **Холм** (z0): один SVG-путь `#DCEBBF`.
- **Фото** (z2): `assets/kids-logo.png`, `left:49%; bottom:34px; transform:translateX(-50%); width:96%`. Меньший масштаб — дети видны почти в полный рост.
- **Белая волна-переход** (z3): SVG-путь `#FFFFFF`, высота 70px, `bottom:-1px` — мягко переводит сцену в белую текстовую часть и аккуратно «обрезает» ноги детей.

**Текст (снизу)** — `background:#FFFFFF; text-align:center; padding:6px 28px 34px`:
- H1: Nunito 600, 27px, line-height 1.18, `#2B2A26`, `margin-bottom:12px`.
- p: Nunito 600, 15px, `#7C7A72`, `margin-bottom:16px`.
- Плашка «Пробные занятия»: как на десктопе, но 13.5px, паддинг `9px 18px`, точка 9×9.
- CTA «Записать ребёнка»: как на десктопе, 18px, паддинг `15px 36px`.

## Interactions & Behavior
- **CTA / плашка**: наведение — рекомендую лёгкое затемнение фона (~6–8%) и/или подъём тени; в прототипе явных hover-состояний нет — примените паттерн кодовой базы.
- **Звёзды**: плавающая анимация `@keyframes bob2 { 0%,100% { transform: translateY(0) rotate(0) } 50% { transform: translateY(-7px) rotate(6deg) } }`, длительности 8–10s, `ease-in-out infinite`. Чисто декоративная, можно опустить.
- **Адаптив**: десктоп-раскладка (2 колонки) переключается на мобильную (сцена сверху, текст снизу) на узких экранах — точка перелома на усмотрение (ориентир ~768px).
- Ссылок/навигации внутри блока нет, кроме CTA (ведёт на форму записи — цель определяется в проекте).

## State Management
Статический презентационный блок — состояние не требуется. Единственное действие — клик по CTA (открыть форму / перейти к записи).

**Тени**: карточка `0 30px 70px rgba(90,110,50,.16)`; CTA `0 14px 30px rgba(124,179,66,.4)`.
**Отступы**: используется шкала кратно ~4px (6, 10, 12, 16, 26, 34, 64).
